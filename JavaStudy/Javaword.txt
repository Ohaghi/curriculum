■ 開発
ウォーターフォール開発...要件定義からコーディング、テストまで上流工程から下流工程まで滝が流れるように業務を行っていくこと。大規模開発に用いられる。
アジャイル開発...小人数の開発に用いられ機能ごとに要件定義を行いその都度改良していくこと。
マイグレーション...データやプログラムを移行すること。
デグレーション...プログラミングを変更した結果前より質が悪くなってしまうこと。
デプロイ...作ったプログラムを使える状態にすること。
コンフリクト...複数のプログラムが一つのデータやメモリにアクセスしてぶつかってしまう状態。
環境...何かプロジェクトなどを行うときに必要な場所、物など
開発環境（ローカル）...開発するときの作業場所、道具など
検証環境（ステージング）...テストの際、実際にシステムを使うことを想定した環境のこと。
本番環境...システムが使われる実際の環境
設計書／仕様書...システムの説明書
要件定義...発注元の求めるシステムの機能を聞き取りまとめる
基本設計書...要件定義の概要をまとめたもの
詳細設計書...プログラムを作る際に実際に見る説明書
テスト...システムが正常に動くか検査する工程
単体テスト（UT）...処理ごとのテスト
結合テスト（CT／IT）...複数の処理を連結して正常に動くか検査するテスト
総合テスト（ST）...システム全体で各処理が正常に動くか検査するテスト
エビデンス／証跡...テストに使ったデータ、テスト結果などテストを行った証拠。
データパッチ...プログラムのバグ改修や機能変更に使うデータのこと
Access...コンピュータやプログラムなどがデータベースにつながること
マクロ...処理や操作手順を記録しておき、まとめて実行できるようにしておく機能
VBA(Visual Basic for Applications)...マイクロソフトが作った、Office ソフト上で動くプログラミング言語
GAS（Google Apps Script）...JSをベースとした、Google開発のプログラミング言語。Googleツール同士、または外部の
　　　　　　　　　　　　　　のツールと連携するために用いられる。
API(Application Programming Interface)...プログラムの外部からその機能を使えるようにするための窓口
WEB API...インターネットを経由するAPI
REST API(Representational State Transfer API)...get/post/delete等で送りxmlで送る
SOAP...xmlで送りxmlで受け取る
Ajax(Asynchronous JavaScript ＋ XML)...JavaScriptの非同期通信によってWebページの一部分だけ変更すること。
XML(eXtensible Markup Language)...マークアップ言語のひとつ。HTMLと違いWebページ以外のたくさんの用途がある。データのやり取りや設定ファイルなど
JSON(JavaScript Object Notation)...JSと相性のいいファイルの書き方の一つ。
ステータスコード...HTTPステータスコードとも呼ばれる。ブラウザがHTMLを受け取る前に受信する三桁の数字。
クロスオリジン（Cross-Origin）／CORS（Cross-Origin Resource Sharing）...別のオリジンをもつデータ間のアクセスを安全に行う仕組み
バリデーション...Webページの入力部分などで、入力した内容が型に合っているかチェックすること。
データ連携...異なるアプリケーションやシステムをまたいでデータを共有、活用するために、それぞれの言葉や解釈の
　　　　　　　垣根を取り払うプロセス
Daemon...Unix系OSに常駐待機しているプログラム。バックグラウンドで動いている。
CRON...Unix系OSに常駐待機しているプログラム。指定した時間にのみ動く。
バッチ...いくつかの処理をひとまとめにした単位
ログ...コンピュータの利用やデータ送受信の記録。
CSV(Comma Separated Values)...値や項目がカンマで区切られている書式
TSV(Tab Separated Values)...値や項目がタブで区切られている書式
プルリクエスト（マージリクエスト）...ローカルリポジトリでの変更を他の開発者に通知する機能
■ チケット管理ツール（タスク管理）
asana...使いやすく初心者向き、IT以外の現場でも利用されている。世界中で最もよく利用される
Backlog...日本でのシェアナンバーワン。バージョン管理ツール機能もある。
Jooto...取引先とのコミュニケーションにも利用可能
Readmine...オープンソースで無料。チケット機能でタスク管理
Trello...仕事のタスクだけでなくプライベートや副業の予定まで管理しチームワークに役立つ。
■ バージョン管理
集中型...1つのリポジトリに複数人が接続する方法
分散型...それぞれがリポジトリを持ち、ローカルで開発できる方法
Github..分散型
Backlog...タスク管理機能もある。
SourceTree...git flowも使える
Tortoise SVN...windowsのシェルと統合されている。
Git Flow...チーム開発でgitをより効果的に使うためのツール。
master...各ブランチの核となる、最初に生成されるブランチ
release...プログラムをリリースするときに使うブランチ。
develop...開発の時に作成し利用するブランチ。
feature...バグ改修、機能変更変更ごとに作られ、再びdevelopにマージするブランチ。
■ FW／ライブラリ
FW(framework)...よく使う機能の処理の流れをまとめたもの。
Spring Framework...webアプリ開発に用いられるjavaの主要フレームワーク
Struts...アプリを作る際の主要javaフレームワーク
JSF（Java Server Faces）...ユーザーアンケートなど、インターフェース作成に特化したフレームワーク。
ライブラリ...プログラムの部品として頻出となるものに名前をつけて呼び出せるようにしているもの。
Bootstrap...cssのフレームワーク
Flyway...データベースのバージョン管理ツール。
Jackson...JSON形式のデータを処理するための高速かつ軽量なJavaライブラリ
jQuery...フロントエンドで用いるJSのライブラリ。
Junit....javaプログラムのテストを自動化するツール。
Log4j...java開発のログを出力するライブラリ。
Lombok...javaのロジックをよりシンプルにするため、定型コードを排除するライブラリ。
ORM(Object/Relational Mapping)....RDBのレコードを、コード上でオブジェクトとして直感的に扱えるようにするフレームワーク
■ 連絡ツール
ChatWork...いつも使ってるやつ。国産。
Slack...アメリカ産。機能が多くIT企業で人気。
IP Messenger...社内LANで利用するメッセンジャーアプリ。
■ Java
Javadoc...自動生成されるjavaソースコードの説明書
アノテーション...コード上に書いて共有するソースコードの注釈。
メモリ管理...新しいオブジェクトのための空き領域を探す、古いオブジェクトを操作して空き領域を作ること。
ヒープ...OSやアプリケーションソフトが使用するメモリ領域の一種。ユーザが作ったプログラムが利用するメモリ領域。
GC(Garbage Collection)...プログラム実行後、不要になったメモリを排除してくれる機能
GoF(Gang Of Four)...オブジェクト指向プログラミングにおける再利用性の高いコーディングのパターン
デザインパターン...ソフトウェア設計者が見つけたノウハウを再利用しやすいように名前を付けてカタログ化したもの
■ その他
オブジェクト指向...一つ一つの処理を機械のパーツのように分け、機能ごとに分類することで改修やコード分析を
		　　しやすくしたもの
バッチ処理...自動化することが可能、かつ緊急でない業務をまとめて行うこと。
コンパイル...人間語に近いプログラミング言語をコンピュータの判断できる言語に翻訳する工程。
カバレッジ...テスト計画に対してどの程度の割合を達成しているかを表す指標。
プラットホーム...プログラムを動かすための土台となる環境のこと。
