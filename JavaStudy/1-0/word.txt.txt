■ IT
IT(Information Technology)-情報技術。ネットワーク、コンピュータを用いた技術。
ICT(Information and Communication Technology)-情報通信技術。ITを人と人のコミュニケーションに活用すること。
IoT(Internet of things)-もの（主に家電など）をインターネットに繋ぎ、遠隔操作を可能にした技術。
■ OS
OS(Operating System)-基本ソフトとも呼ばれ、メモリの管理やハードウェアの制御等、
　　　　　　　　　　コンピュータの動作にかかわる働きをするソフトウェア
Windows-1985年にMicroSoftから発売されたOSブランド。
macOS (Mac OS X)-1984～2001年に開発されていたClassic Mac OSの後継的なOS。2001年より使用技術が大きく変化した。
UNIX-ほぼ最古のOSであり、MacOSのベースとなった。ライセンスをかけたため料金やその他制限が多い。
Linux-UNIXをもとにつくられ、無料・オープンソースで提供されており企業のサーバーとして人気。
■ 言語・環境
マークアップ／フロントエンド-インターネット上で目に見える部分のこと。
　　HTML(HyperText Markup Language)-マークアップ言語の一つ。Webサイトのほとんどに使用されているが単独では
　　　　　　　　　　　　　　　　デザインを操作できないためCSS等と併せて利用する。
　　CSS-HTMLと組み合わせて使うことでWebサイトの色や配置等のデザインを操作することができるマークアップ言語。
　　JavaScript-フロント・バックエンド両方で動く言語。JQuery等、WEbサイトの見た目に動きをつける機能がある。

サーバーサイド-バックエンドともいう。フロントエンドに対してユーザーの目には見えない機能の処理を行う部分。
　　　　　　　 サーバーとのやりとりなど
　　.NET（C#, VB）-Windows系アプリケーションを作成できるサーバーサイド言語。
　　Java-Web,アプリやゲーム等、どの形式のプログラムにも対応できる言語。業務システムでの需要が高い。
　　Java SE(Java Plattform Standard Edition)-Javaで使用されるAPI(Application Programming Interface)をまとめたもの。デスクトップアプリケーション等
　　　　　　　　　　　　　　　　　　　　　の開発で一番よくつかわれる。
API(Application Programming Interface)-Javaの機能やデータなどを利用するための呼び出し方を定義したもの。
Java EE（Jakarta EE）-Java SEに対して、大規模な開発に頻繁に用いられる。
Perl-言語の一種でC言語等から影響を受けている。gitやmysqlで利用されている。
PHP-Webページ作成で頻繁に用いられるサーバーサイド言語
Ruby-日本人によって開発されたサーバーサイド言語。ruby on rails等のフレームワークが発達していることにより
　　 開発スピードの求められる現場で多く用いられる。
■ サーバーソフト-サーバー機能を提供するソフトウェアの総称。単に「サーバー」とよばれることも多い。
Apache（アパッチ）-代表的なWebサーバーソフトウェアの一つで、ウェブサービスの半数のシェアを占める。
　　　　　　　　　 オープンソースソフトウェアで、無償で誰でも利用できる。
nginx（エンジンエックス）-Apacheに比べてスピードが速く、複数アクセスにも強い。Apacheに次いで二番目によく
　　　　　　　　　　　　　使用されるサーバーソフト。
IIS(InternetInformationService)-WindowsOS上で操作でき、コマンドを使わずに視覚的に操作ができる。
■ SQL
SQL（シークエル）-データベース言語のひとつ。ISO（国際標準化機構）で規格が標準化されているため、他の
　　　　　　　　　データベースを利用する際も汎用性がある。
　　DB2-IBM社のリレーショナルデータベース管理システムで、データベース言語はSQL。企業の基幹システムでの使用頻度
　　　　が高い。
　　リレーショナルデータベース管理システム(RDBMS)-複数の行と列から成るテーブルでデータ間の関係を表すデータベース。
　　MySQL-オープンソースのRDBMS。検索スピードに優れておりyahoo,googleなど多くのサービスで利用されている。
Maria DB-MySQLから派生したオープンソースRDBMS。Wikipedia,Googleなどに利用されており、近年シェアを急速に
　　　　 拡大している。
　　Oracle DB-Oracleが開発販売しているRDBMS。同時複数リクエストに強い。
　　PostgreSQL-オープンソースのRDBMS。導入コストが低いのが特徴。
　　SQLite-MySQLやPostgreSQLのように同時複数リクエストに対応するが、ローカルファイルを直接データベースとして
　　　 　　扱うため複数のサーバーを使うような開発に不向き。
　　SQL Server-正式名称はMicrosoft　SQL Server、Microsoftが開発したRDBMS。WindowsOS上で操作でき、コマンドを使わずに
           　　視覚的に操作ができる。(GUI)
CRUD(Create、Read、Update、Delete)-データベースにおける登録・参照・変更・削除機能の総称。
トランザクション-複数の処理を一つにまとめること。
　　コミット-処理を確定させる、反映させる意味。
　　ロールバック-データ更新などで障害が起こったときに、その前の状態にまで戻ること。
排他制御（排他ロック）-データベースなどで同時アクセスによる不整合が起こらないように設定すること。
　　楽観ロック-そう頻繁に他社との同時更新は起きないだろうという楽観的な前提条件で設定された排他ロック。
　　悲観ロック-常に同時更新のリスクが起こる可能性があると悲観的な前提条件で設定された排他ロック。
実行計画-テーブルに対する検索方法が記された、SQLの実行手順書。
チューニング-データベースのパフォーマンス向上のため、速度やメモリ容量等の調節を行うこと。
パフォーマンス-システムの処理性能、実行速度のこと。
スロークエリ-呼び出し等に時間のかかるSQL。
クエリログ-実行されたSQLの記録。
IDE（統合開発環境）-テキストエディタ、コンパイラ、デバッガ等、開発に必要なアプリケーションをまとめた
　　　　　　　　　　ソフトウェアパッケージ。
　　Eclipse-オープンソフトウェアとして公開されているIDE。Javaに多く用いられるが、PHP、C#、Ruby等にも対応している。
　　NetBeans-Eclipse同様。Javaの開発を目的に作られたが、現在では他の言語にも利用されている。
　　SpringToolSuite-Eclipseをベースとして作られたIDE。JavaのSpringフレームワーク。
DBツール
　　A5:SQL Mk-2-データベース開発において生産性を向上させるべくたくさんの機能を持ち合わせたSQL開発ツール。
　　pgAdmin-GUIによるPostgreSQLの管理ツール。
　　DBeaver-ほぼすべてのデータベースに対応したデータベースツール。
　　MySQL Workbench-データベース管理を視覚的に行えるツール。
　　Sequel Pro-データベースをExcelのような表で確認・編集をすることができるMac専用のGUIアプリケーション。
　　TeamSQL- 全てのDBで利用できるDBツール。クエリログが残る、開発チーム内でSQLの共有が簡単に行えることが特徴。
■ 肩書
PM（プロジェクトマネージャー）-プロジェクトの運営、品質、納期等に責任を持ちプロジェクトを円滑に推進する役割。
PL（プロジェクトリーダー）- システム開発において、プロジェクトチームを管理・統括する役割。プロジェクトにつき
　　　　　　　　　　　　　　一人のPMにつきPLは複数人いる。
PG（PE）（プログラマー）-　システム開発において、SEの仕様書に沿って実際にコードを書きながらプログラムを作る役割。
SE-　開発チームの管理、プロジェクトの仕様書、システムの設計図作成を担う役割。
デザイナー-システムやWebサイトの視覚デザインを手掛ける役割。
エンジニア-ネットワークシステムの設計や構築、運用や保守を行う職業。
　　フロントエンド・エンジニア-システムのうちクライアント側で操作する部分を開発するエンジニア。
　　サーバーサイド・エンジニア-システムのうちサーバー側で操作する部分を開発するエンジニア。
　　バックエンド・エンジニア-上記のエンジニアの同義語。
■ UI
UI（ユーザーインターフェース）- コンピュータとその機械の利用者（通常は人間）の間での情報を
　　　　　　　　　　　　　　　　やりとりするためのインタフェース
インターフェース - 境界面、接触面。IT用語ではコンピューターと周辺機器またはコンピュータと人の接続部分を意味する。
　　CUI(キャラクターUI)- コンピュータやソフトウェアが利用者に情報を提示したり操作を受け付けたりする方法で、
　　　　　　　　　　　　 文字を媒体にしている。
　　GUI-(グラフィカルUI)- 画面のタッチやスクリーン上でのクリック＆ドロップ等、視覚的に操作する。
■ ファイル／ディレクトリ
ファイル-コンピュータにおけるデータ管理の単位の一つ。複数のデータを一つのまとまりとして表す。
拡張子-「sample.txt」等のファイルの「txt」の部分。ファイルの形式や種類を表す役割がある。
ディレクトリ-ファイルシステムなどで、ファイルを分類・整理できる保管場所。（フォルダ）
　　ルート・ディレクトリ-ファイルシステムにおいて、装置やシステム全体の最上位のディレクトリのこと。
　　カレント・ディレクトリ-現在作業中のディレクトリのこと。
パス-主にファイルやディレクトリの位置を表す文字列のこと。
　　絶対パス-ルートディレクトリから目的のファイルまで全てのディレクトリを記述するパスの方式。
　　相対パス-カレントディレクトリからの相対的な位置関係で表すパスの方式。
■ ネットワーク
LAN(local area network)- 限られた範囲内にあるコンピュータや電子機器をケーブルや無線でつなぎ、
　　　　　　　　　　　　 相互のデータ通信を可能にするもの。
WAN(wide area network)- 地理的に離れた地点の機器同士を結ぶ通信ネットワーク。
クライアント-情報の提供を受ける側のコンピュータやソフトウェア。
サーバー-情報の提供をする側のコンピュータ。
■ 各種プロトコル-複数の端末が滞りなく信号やデータを相互に送るためのルール。
HTTP(HyperText Transfer Protocol)- サーバとクライアント間でのデータ、特に文字や画像、動画の送受信を行うためのプロトコル。
FTP(File Transfer Protocol)-サーバとクライアント間でファイルを送受信するためのプロトコル。
HTTPS(HTTPsecure)- HTTPに暗号化の機能が加わり、より安全性の高いもの。
SSL(Secure Sockets Layer)- ブラウザとサーバ間のデータ通信を暗号化し、送受信するための仕組み。
SSH(Secure Shell)-クライアントから遠く離れたサーバを遠隔操作し、管理するための手段
ポート、ポート番号-TCP/IP通信において、 コンピュータが通信に使用するプログラムを識別するための番号。
HTTP、FTP、SSHの各ポート番号-80   21   22
ファイアウォール-企業などの内部ネットワークをインターネットを通して侵入してくる不正なアクセスから
                 守るための“防火壁”、セキュリティシステム。
TCP(Transmission Controll Protocol)-送信したデータが相手に届いたか確認する安全性重視のプロトコル。
■ IPアドレス
IPアドレス(Internet Protocol address)-コンピュータごとにあてられている住所のようなもの
　　グローバルIPアドレス-全世界で通用する、重複することのないIPアドレス。
　　プライベートIPアドレス-ローカルネットの範囲内で決めるipアドレス。
■ ドメイン
ドメイン-インターネット上に存在するコンピューターやネットワークを識別するための名前、...@gmail.com,  /www.(ドメイン)
DNS(Domain Name System)- IPアドレスとドメイン名を紐づけ、変換する仕組み。
VPN(Virtual Private Network)-公共のネットワークを私的なネットワークのように使い安全性を高めるサービス。
ブリッジ-ネット上のデータのmacアドレスから、データの行先を振り分ける機器。
プロキシサーバー-クライアントとサーバのデータやり取りの間に入る代理サーバ。クライアントの身元を隠せる利点がある。
■ 電子メール
SMTP(Simple Mail Transfer Protocol)-メール送信に使われるプロトコル
POP(Post Office Protocol)-メールサーバに届いたメールをコンピュータ上で表示するのに使うプロトコル。
IMAP(Internet Message Access Protocol)-上に同じだが、メールサーバからメッセージをコピーしてきて表示するプロトコル。
■ セッションとクッキー
セッション-通信における最初から最後までのひとまとめの単位。
クッキー-サーバを通してwebページを見るときにサーバからブラウザに渡すメモのようなもの。
キャッシュ-web上でよく使われるデータをすぐ呼び出せるように準備しておく仕組み。
■ DB（データベース）について
インデックス-データベース内のデータ検索のスピードを上げる機能
■ テストについて
フロー-開発テストの流れ。

タスク-それぞれのエンジニア、もしくはコンピュータに課される仕事
■ バージョン管理
Git-バージョン管理システムであり、ファイルの変更を管理する。
SVN(Apache Subversion)- gitに同じ。
デグレ（デグレード）-　プログラムを変更した際に、改悪されてしまうこと。
マージツール-新たにコードを追加したり、修正を反映させたりする操作するためのツール。
